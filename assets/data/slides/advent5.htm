Making it useful. Defining some routes

  • Routes are the main building blocks of a sammy application
  • A route is made up of a verb, a path and a callback function
  • The verb can be one of: get, post, put or delete. Of these get is the most commonly used because...
  • post, put and delete are only invoked on form submission
  • Within the callback of a route this is a new Sammy.EventContext. This has a number of handy methods and attributes like redirect or render
