Events. Defining, binding and triggering

  • Events in sammy are designed to be created in the same way as in jQuery
  • Internally events are bound to the application within a unique application namespace
  • This means you can run multiple sammy applications without having to worry about multiple events being triggered
  • bind creates a handler for an event
  • trigger causes an event to be raised, can be raised within the context of the application or within other events/routes
